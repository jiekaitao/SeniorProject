Core Thesis

LLMs process text left-to-right in a single pass — they lack iterative internal computation. We prove that a 12M-parameter bidirectional solver bolted onto a frozen Llama 3.1 8B doubles spatial reasoning accuracy on a NeurIPS 2024 benchmark. Crucially, scaling alone fails: Gemma 4 31B performs worse (24.5%) than our augmented 8B system (70.6%).

Three Lines of Evidence

1. External Solver Architecture: A bidirectional "thinker" produces memory tokens for the frozen "talker" LLM. Bypass mode (solver augments, doesn't replace input) is critical.
2. TRM Mechanistic Analysis: Standard BCE + StableMax creates an overconfidence trap — 98.5% confident but 80.6% correct, frozen at a wrong fixed point. Brier + monotonicity + softmax breaks the trap via an 18.1pp synergistic interaction.
3. Runtime Layer Duplication: Duplicating attention blocks at inference yields +7.31 over Ng's result on 72B. FFN duplication hurts — attention-only is the key.

1. Reproduce Ng's RYS Result

March 15, 2026 · Qwen2-72B (80 layers)

Confirmed: Layer duplication improves reasoning

Duplicating layers [45,52) of calme-2.1-qwen2-72b reproduces Ng's result: combined score 76.76 vs 70.52 baseline (+6.24). The mechanism: the first pass computes $h \to F(h)$, the second pass computes $F(h) \to F(F(h))$, acting as iterative refinement with shared weights.

$$h^{(2)} = F(F(h^{(0)})), \quad \rho_{\text{disp}} = \frac{\|F(F(h)) - F(h)\|}{\|F(h) - h\|} < 1$$

Displacement ratio $\rho < 1$ means the block converges when repeated — each pass moves the representation closer to a fixed point.

2. Spectral Screening Framework

March 16–18 · 7B + 72B models

Predicting good blocks without brute-force search

Ng tested 3,241 configurations by brute force. We developed spectral metrics that predict which blocks benefit from duplication:

$$\text{SBUID}_0 = \text{BLOOD}_{\text{impact}} - 6000 \cdot \rho_{\text{perturbation}}$$

Achieves Spearman r=0.515, p=0.008 on 72B — statistically significant block ranking from a 5-minute computation instead of days of evaluation.

3. Junction Adapters (Failed)

March 18–20 · 7B model

Every trained adapter hurts good configurations

Tried 6 adapter approaches (LoRA, BLOOD-guided, junction smoothing) at the seam between first and second pass. All degraded the best duplication configs. The untrained duplication works because it reuses exact weights — any learned perturbation breaks the fixed-point convergence.

5. Per-Layer Alpha Tuning

March 22 · 72B, 300 grid evaluations

Best result: +7.31 over Ng's published score

Instead of $h_{\text{out}} = F(F(h))$, blend with per-layer alphas:

$$h_{\text{out}}^{(\ell)} = h_1^{(\ell)} + \alpha_\ell \cdot (h_2^{(\ell)} - h_1^{(\ell)})$$

Optimized via grid search (300 evals) and Bayesian optimization (60 evals). Triple-block per-layer alpha achieves 84.07 combined (Ng: 76.76).

8. Sublayer Analysis: FFN Hurts

March 27

FFN re-execution "overshoots" stored facts

Attention-only duplication preserves the reasoning benefit while avoiding FFN re-retrieval corruption. This is why MATH degrades: the FFN stores factual associations that get distorted by double execution.

9. KV Cache Fix

March 28

LayerIdxWrapper solves shared-module KV indexing

Deep-copied layers with unique layer_idx + layer_types=None in config allow proper KV caching for duplicated models.

10. Trained Recursion (Failed)

March 29

Verdict: only untrained duplication helps

LoRA fine-tuning for multi-pass showed no improvement. Training the model to "re-read" doesn't work — the benefit comes from the raw mathematical properties of applying the same function twice, not from learned second-pass behavior.

PSRT: From-Scratch Recursive Model

Projected Split-State Recurrent Transformer trained from scratch. Neither v1 (fineweb-edu only) nor v2 (50% general + 25% math + 25% science) showed K-scaling benefits. The recursive mechanism needs to be bolted onto an existing pretrained LLM, not trained from scratch.

The Overconfidence Trap

April 7 · Diagnostic analysis on 200 mazes

BCE + StableMax → confidently wrong, permanently frozen

MetricFULL_COMBOBASELINEMeaning
Step 0 confidence96.4%98.5%BL more confident
Step 0 reach acc98.1%80.6%...but less accurate
Step 1 reach acc100%78.7%FC done; BL stuck
cos(z_H step 0→1)0.8920.964FC still refining; BL frozen

The mechanism: BCE's log singularity creates near-zero gradients for confidently-wrong predictions. StableMax has a discontinuous second derivative at $x = 0$:

$$s(x) = \begin{cases} \frac{1}{1-x} & x < 0 \quad (\text{curvature} = \frac{2}{(1-x)^3}) \\ x + 1 & x \geq 0 \quad (\text{curvature} = 0) \end{cases}$$

This Hessian ridge + BCE log singularity + shared weights across 16 ACT steps = the model converges to a wrong fixed point and can't escape. Brier loss ($L = (\sigma(z) - y)^2$, bounded, smooth) breaks the trap.

Solver doubles accuracy: "Separate the thinker from the talker"

In bypass mode, the frozen decoder sees [memory tokens | full prompt]. The bidirectional solver compresses spatial structure into 32 memory tokens that the causal LLM uses as a "cheat sheet."

$$p(y|x) = F_\omega(y \mid [\underbrace{M_\theta(x)}_{\text{solver memory}};\ x])$$

The solver ($M_\theta$, 12M params) is bidirectional — it can see the full maze simultaneously, unlike the causal decoder ($F_\omega$, frozen 8B).
