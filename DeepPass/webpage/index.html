The core finding At matched 140M parameter budget, a recurrent deliberation controller beats LoRA fine-tuning on SpatialGrid by more than +20pp (base) and is stable across all seeds, while uniquely offering test-time K-scaling. LoRA still wins on semantic benchmarks (WinoGrande, HellaSwag, Mazenav). We identified a reproducible fine-tuning failure mode on structured reasoning: 3 out of 10 LoRA SpatialGrid runs collapse below random.

Controller architecture

A small (~140M) learned controller writes thought tokens into the residual stream of a frozen Llama 3.1 8B. It reads hidden states tapped at layers {8,16,24}, updates its own state z ∈ ℝ8×512, and splices the new thoughts back in at layer 12 for another pass. Same weights, K iterations.

[prompt] layers 0..11 [prompt|thoughts|answer] layer 12..31 ↓ ↓ ↓ ↓ embed tokens → frozen LM layers → splice thoughts → frozen LM layers → ABCD logits ↑ │ z ∈ R^(8×512) —write—┘ │ ↑ │ RMSNorm(z + σ·Δ) ←— read features ←——————————————————┘ (tapped layers 8,16,24 + slot hidden states) [iterate K rounds]

The write operation mixes a sparse vocabulary-distilled signal with a low-rank residual:

$$\text{thought}(z) = \sum_{k=1}^{64} \text{softmax}(\text{logits}_k(z)) \cdot E[\text{topk}_k(z)] + 0.12 \cdot z\, W_\downarrow W_\uparrow$$

Honest assessment

This is a narrow capability, not a general intelligence improvement. The controller loses to LoRA on WinoGrande (+10pp vs +19pp), HellaSwag (+12pp vs +14pp), CommonsenseQA (+2pp vs +9pp), and Mazenav (+0pp vs +73pp). Wins are specific: (a) SpatialGrid, where the task requires per-instance operation execution that LoRA cannot memorize; (b) vision on PaLiGemma. The contribution is identifying when iterative computation helps, not a universal upgrade. DeepPass duplication similarly has a sharp profile: it helps reasoning (IFEval +2.3%, MuSR +1.3%) but hurts knowledge (MATH −6.4%).
